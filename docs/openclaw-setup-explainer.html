openclaw-setup v1.0

Repeatable OpenClaw deployment with secure secrets and atomic config

[!] THE PROBLEM

Manual Setup

  • Agents edit openclaw.json directly -- JSON corruption (Bug #6028)
  • Gateway overwrites config on startup (#11355)
  • API keys pasted into config end up in git history
  • Setting provider fields one-by-one fails validation
  • Every install is different -- no repeatability

openclaw-setup

  • All config via openclaw config set -- no direct JSON
  • Secrets in ~/.openclaw/.env with chmod 600
  • Atomic provider definitions via --json
  • Non-interactive mode for CI/fleet with OC_* env vars
  • Same script for fresh install, recovery, and fleet

[*] WHY USE IT

{} No JSON Corruption

All config changes go through openclaw config set. The CLI validates schema before writing.

@ Secrets Never in Config

API keys stored in ~/.openclaw/.env. Config uses ${VAR} references. chmod 600.

# Atomic Providers

Custom model providers set as complete JSON blocks via --json. No partial schema states.

=> Repeatable

Same script for fresh install, disaster recovery, and fleet deployment. --non-interactive for CI.

[>] COMMANDS
bash scripts/openclaw-setup.sh Interactive setup with prompts
bash scripts/openclaw-setup.sh --non-interactive CI/fleet mode using OC_* env vars
bash scripts/openclaw-setup.sh --dry-run Print commands without executing
bash scripts/openclaw-setup.sh --skip-install Skip npm install step
for t in scripts/test/test-T*.sh; do bash "$t"; done Run all 15 test suites
openclaw config set <path> <value> Safe config changes (NEVER edit JSON directly)
